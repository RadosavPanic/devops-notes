# Key cloud terminology

## Elasticity

Peak Load Provisioning is providing enough servers to handle peak times (holidays, sales, weekends) - maximum usage. This ensures that sites don’t crash when they’re mostly needed due to large number of uses.

> [!IMPORTANT]
> Elasticity is ability to automatically scale resources up or down based on demand.

- **Scale on demand**: Add more servers instantly when traffic peaks
- **Release servers**: Release servers when traffic stops
- **Impact**: No cost needed for idle resources

## Agility

Earlier, businesses had to build their own infrastructure - data centers which costs large amount of money beforehand and takes time to buy and set up new servers. At this point, innovation was being slowed down due to infrastructure build process.

Cloud solution provides infrastructure ready for businesses with high security.

> [!IMPORTANT]
> Agility is ability to move fast and respond quickly to business needs. It allows businesses to quickly deliver software and quickly provision of infrastructure.

## Global Reach and Geo-Distribution

To be able to deliver good customer experiences earlier, companies would need to build data centers around the world. This would prevent high latency for global users, as servers would be located in the nearby region.

With Cloud, deploy to multiple regions is available in minutes.

- **Global Reach**: Expand to new markers around the world
- **Better experience**: Provide low latency - faster access to users worldwide
- **Lower cost**: No need to build data centers and manage infrastructure in multiple locations

> [!IMPORTANT]
> Geo-Distrubution is ability to expand to new markets and deploy applications to data centers around the world quickly.

## Availability

Availability is ability to keep applications available when users need them.

## Latency

Latency is time taken for a request to travel between user and server.

It is reduced by serving users from nearby locations.
